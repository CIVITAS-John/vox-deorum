# v0.9.0 — 2026-04-02

Updated Vox Populi to 5.2.2 (Beta). Savegame incompatible.

## Archivist (New System)
- Full **Archivist** pipeline for learning from past games — scans episodes, extracts key moments, transforms data, and writes to DuckDB
- Five-stage pipeline: scanner, extractor, game-state vectorizer, landmark selector, and writer with embeddings
- **Episode retrieval** system with similarity search for finding relevant past experiences
- Multi-archivist and multi-thread support for concurrent processing
- Telepathist preparation integration — archivist feeds historical context into live games
- DuckDB-based episode storage with schema and plan documentation
- Per-population metrics (gold/culture per pop) for better cross-game comparisons
- Console interface for running the archivist standalone

## Strategist
- New **null-strategist** that forces NULL strategy onto tactical AI (useful for benchmarking)
- New **learned strategist** that leverages archivist episodes to inform strategic decisions
- Improved strategy session handling and parameter management
- Refactored simple-strategist base for better code reuse

## Telepathist
- **Episode retriever** integration — retrieves relevant historical episodes during gameplay
- Major refactor of preparation system into modular phase/turn preparation with dedicated instructions
- New `get-situation` tool replaces `get-game-overview` and `get-game-state` (consolidated)
- Renamed `get-decisions` → `get-decision` with improved implementation
- Telepathist no longer requires MCP server to run (standalone mode)
- Reduced context window issues and token usage
- Improved summarizer and parameter handling

## MCP Server
- New `get-game-settings` tool (split from `get-metadata` for clarity)
- Reshaped `DealMade` events for better LLM consumption
- Better feedback messages for tech/policy action tools
- Expanded event visibility in Lua scripts
- Grand flavor strategy support
- Improved search weights and database management
- Stop removing invalid tool calls from downstream

## Bridge Service
- Comprehensive new test coverage: reconnect state, pause manager, DLL connector buffering, external manager, Lua manager, bridge service integration
- Improved DLL connector with better buffering
- Updated external manager with minor fixes
- Enhanced test infrastructure with isolated mocks and real bridge test support
